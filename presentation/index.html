Testing Terminology

  • Lots of testing terms - unit vs integration, etc.
  • This presentation will cover my own term, "Automated Developer testing"
  • Developer Testing - automated tests run often by Developer, while coding, typically from their IDE.

Traditional Testing

  • MVC controllers are C# classes
  • In a test, construct the controller, test the action methods.

Traditional Testing - Problems?

  • Misses anything that happens before or after the action method of controller.
  • Action or controller attributes - e.g. Routing, Authorize attributes
  • Response codes, response formatting
  • Request parsing
  • Dependency Injection configuration

New ASP.Net Core Testing Features

  • 1.x - TestServer
  • 2.x - WebApplicationFactory - helper method around TestServer
  • TestServer - in-memory version of entire ASP.Net Core Server, with connected HttpClient instance.
  • Full stack executes, middleware, routing, authorization, model binding, DI, etc.
